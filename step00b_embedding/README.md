# step00b — Embedding：向量空间

## 教学目标

理解 token_id → 向量的查表过程，以及为什么用可学习向量而不是 one-hot。

## Embedding 矩阵结构

```
                 d_model=8
            ┌─────────────────┐
token_id=0  │ 0.23 -0.11 ...  │  ← 字节 '\x00' 的向量
token_id=1  │ 0.87  0.45 ...  │
   ...      │      ...        │
token_id=65 │ 0.34 -1.21 ...  │  ← 'A' 的向量  ←── embed('A')
   ...      │      ...        │
token_id=255│ -0.56 0.33 ...  │
            └─────────────────┘
             vocab_size=256 行
```

查表操作：`output = weight[token_id]`，就是取矩阵的第 token_id 行。

## 为什么不用 one-hot？

```
one-hot('A') = [0, 0, ..., 1, ..., 0]   ← 256维，全0只有1个1
                              ↑ 第65位
问题：
  1. 高维稀疏，浪费计算
  2. 任意两个词距离完全相同（都是 √2），无法表示语义关系

Embedding('A') = [0.34, -1.21, 0.87, ...]  ← 8维（d_model）
优点：
  1. 低维稠密
  2. 训练后语义相近的词向量接近：sim('king','queen') ≈ 0.97
```
